# Bible SumOne - 핵심 기능 브레인스토밍

**작성일**: 2026-03-24
**목적**: PM/Designer/Engineer 관점에서 핵심 기능 아이디어 도출 및 우선순위 결정

---

## 제품 컨텍스트

- **타겟**: 20-30대 크리스천 커플 (연애/약혼/신혼)
- **핵심 가치**: Together but Async, Bite-sized but Consistent
- **컨셉**: SumOne의 재미있는 UX + 성경의 깊이
- **기술 스택**: Flutter, Supabase, 성경 API, AI API (Gemini/GPT)

## 1. Product Manager 관점 (시장 적합성 & 가치 창출)

### PM-1: AI 기반 일일 큐레이션 시스템 ⭐⭐⭐⭐⭐
**설명**: Gemini API를 활용하여 커플의 상황(연애/약혼/신혼)과 계절/절기에 맞는 성경 구절 + 대화 질문을 자동 생성

**가치**:
- 매일 신선하고 관련성 높은 콘텐츠 제공
- 수동 큐레이션 비용 절감
- 개인화된 경험으로 사용자 만족도 증가

**핵심 가정**:
- AI가 생성한 질문이 신학적으로 적절하고 커플 대화에 적합할 것
- 사용자들이 AI 생성 콘텐츠를 신뢰할 것

**우선순위**: P0 (MVP 필수)

---

### PM-2: 스트릭 & 마일스톤 시스템
**설명**: 함께 읽은 연속 일수, 총 말씀 개수, 특별한 날 (50일, 100일, 1년) 축하 기능

**가치**:
- 습관 형성 동기 부여
- 게임화로 재미 요소 추가
- 장기 리텐션 증가

**핵심 가정**:
- 스트릭이 사용자들에게 실제 동기부여가 될 것
- 100일 이상 지속하는 커플이 10% 이상 될 것

**우선순위**: P0 (MVP 필수)

---

### PM-3: 주제별 컬렉션 (사랑/용서/소통/기도)
**설명**: 특정 주제에 대해 일주일 동안 집중적으로 읽을 수 있는 큐레이션 콘텐츠

**가치**:
- 특정 이슈가 있는 커플에게 도움 제공
- 콘텐츠 깊이 증가
- 교회/목회자 추천 가능성

**핵심 가정**:
- 커플들이 특정 주제를 선택해서 읽고 싶어할 것
- 주제별 접근이 무작위 접근보다 효과적일 것

**우선순위**: P1 (출시 후 1개월 내)

---

### PM-4: 커플 초대 & 매칭 시스템
**설명**: 한 사람이 가입 후 파트너를 초대하고 커플 계정을 연결하는 온보딩 플로우

**가치**:
- 바이럴 성장 (초대 기반)
- 커플 단위 사용으로 이탈률 감소
- 네트워크 효과

**핵심 가정**:
- 초대받은 파트너의 70% 이상이 가입할 것
- 커플 중 한 명이 이탈하면 다른 한 명도 이탈할 가능성 높음

**우선순위**: P0 (MVP 필수)

---

### PM-5: 기도 제목 공유 기능
**설명**: 서로를 위한 기도 제목을 공유하고 기도했음을 표시하는 기능

**가치**:
- 영적 친밀감 증가
- 일상적인 소통을 넘어선 깊은 연결
- 차별화 요소 (다른 커플앱에 없음)

**핵심 가정**:
- 커플들이 기도 제목을 나누는 것을 편안하게 느낄 것
- 주 1회 이상 기도 제목을 공유할 것

**우선순위**: P1 (출시 후 1개월 내)

## 2. Product Designer 관점 (사용자 경험 & 인게이지먼트)

### D-1: Dual Reveal UX (동시 공개 애니메이션) ⭐⭐⭐⭐⭐
**설명**: 두 사람이 모두 답변을 완료하면 카드가 뒤집히듯 서로의 답변이 동시에 공개되는 애니메이션

**가치**:
- 기대감과 설렘 제공 (SumOne의 핵심 UX)
- 감성적인 경험
- 차례로 보는 것보다 "함께 공개"의 느낌

**핵심 가정**:
- 동시 공개가 차례대로 보는 것보다 더 좋은 경험일 것
- 애니메이션이 지나치게 느리거나 빠르지 않을 것

**우선순위**: P0 (MVP 필수)

---

### D-2: 온보딩 스토리텔링 플로우
**설명**: "두 사람의 신앙 여정을 함께 시작해요" 컨셉의 감성적인 온보딩

**가치**:
- 첫인상 강화
- 제품 가치 명확 전달
- 감성적 연결로 이탈률 감소

**핵심 가정**:
- 온보딩이 너무 길면 이탈할 것 (3단계 이내)
- 감성적인 메시지가 20-30대에게 효과적일 것

**우선순위**: P0 (MVP 필수)

### D-3: 상대방 답변 대기 시 "기대되는 순간" UI
**설명**: 상대방이 아직 답변하지 않았을 때 "○○님이 답변 중이에요" 같은 playful한 메시지

- 파트너에게 알림/독려 효과

**핵심 가정**:
- 대기 UI가 답변률을 높일 것
- 너무 잦은 알림은 스팸으로 느껴질 수 있음

### D-4: 다크모드 & 성경책 느낌의 타이포그래피
**설명**: 밤에 읽기 편한 다크모드, 성경책처럼 정갈한 세리프 폰트 옵션

**핵심 가정**:
- 사용자의 50% 이상이 저녁/밤에 사용할 것
- 세리프 폰트가 가독성을 해치지 않을 것

**우선순위**: P1 (출시 후 1개월 내)

## 5. AI 기능 통합 전략

### 5.1 Gemini API 활용

**기능**: 일일 질문 자동 생성
**입력**: 성경 구절 + 커플 메타데이터 (연애/약혼/신혼)
**출력**: 3개의 대화 질문

**Prompt 예시**:
```
당신은 크리스천 커플을 위한 성경 공부 가이드입니다.
다음 성경 구절을 읽은 [연애 중인] 커플이 서로 나눌 수 있는 대화 질문 3개를 생성하세요.

성경 구절: [고린도전서 13:4-7]
"사랑은 오래 참고 사랑은 온유하며..."

요구사항:
1. 커플의 관계에 적용 가능해야 함
2. 너무 무겁지 않고 자연스러운 대화 유도
3. 한 사람만 답하는 것이 아니라 "서로" 나눌 수 있는 질문

출력 형식:
1. [질문1]
2. [질문2]
3. [질문3]
```

**비용 예상**:
- Gemini 1.5 Flash: $0.075 / 1M tokens (입력), $0.30 / 1M tokens (출력)
- 일일 1,000 커플 × 300 tokens ≈ $0.12/day = **$3.6/month**

---

### 5.2 GPT-4 API 활용 (선택적)

**기능**: 답변 인사이트 생성 (P2)
**입력**: 두 사람의 답변
**출력**: 공통점, 차이점, 대화 포인트 요약

**Prompt 예시**:
```
다음은 크리스천 커플이 같은 성경 구절을 읽고 나눈 답변입니다.

질문: "이 말씀을 우리 관계에 어떻게 적용할 수 있을까요?"

A의 답변: "서로에게 더 인내심을 가져야 할 것 같아. 요즘 바빠서 짜증낼 때가 있었거든."

B의 답변: "나도 상대방이 힘들 때 더 이해하려고 노력해야겠어. 사랑은 자기 뜻대로 하지 않는다는 말이 와닿았어."

이 두 답변의 공통점, 차이점, 그리고 더 깊은 대화를 위한 포인트를 한국어로 요약하세요.
```

**비용 예상**:
- GPT-4o: $2.50 / 1M input tokens, $10 / 1M output tokens
- 선택적 기능으로 제공 (프리미엄?)

---

### 5.3 성경 API 통합

**선택된 API**: [bible.helloao.org](https://bible.helloao.org/)
**이유**:
- ✅ 완전 무료, API 키 불필요
- ✅ 1000+ 번역본 (한국어 개역개정, 영어 NIV/ESV 등)
- ✅ JSON 형식
- ✅ 무제한 사용

**API 엔드포인트 예시**:
```
GET https://bible.helloao.org/api/[version]/[book]/[chapter]:[verse]

예: https://bible.helloao.org/api/KRV/John/3:16
```

**캐싱 전략**:
1. 첫 요청 시 Supabase에 저장
2. 이후 로컬 DB (Hive)에서 조회
3. 7일 후 캐시 만료 (선택적)
